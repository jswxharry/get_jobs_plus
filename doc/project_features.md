# Get Jobs Plus - 项目功能总结

## 🎯 项目概述
Get Jobs Plus 是一个基于 Spring Boot + Next.js + Playwright 的自动化智能求职系统，帮助用户在多个招聘平台自动执行岗位搜索、筛选、投递和状态跟踪。

## ✨ 核心功能

### 1. 多平台自动化投递
- **Boss直聘** - 主要推荐平台，支持AI智能打招呼
- **猎聘** - 推荐平台，打招呼无上限
- **前程无忧(51job)** - 支持自动化投递
- **智联招聘** - 支持自动化投递

### 2. AI智能功能
- **智能匹配** - AI检测岗位匹配度，自动筛选合适职位
- **智能打招呼** - 根据岗位描述(JD)自动生成个性化打招呼语
- **图片简历** - 自动发送图片简历，提高HR回复率

### 3. 智能过滤系统
- **黑名单过滤** - 自动过滤已投递或不适合的企业
- **薪资过滤** - 根据目标薪资范围筛选职位
- **HR活跃度过滤** - 自动过滤不活跃HR
- **猎头岗位过滤** - 可选择性过滤猎头发布的职位

### 4. 图形化管理界面
- **任务管理** - 创建、监控和管理投递任务
- **配置管理** - 集中化配置AI密钥、Cookie、筛选条件
- **实时日志** - 通过SSE实时查看投递进度和结果
- **结果分析** - 查看投递成功率和岗位分析报告

### 5. 通知与监控
- **企业微信推送** - 实时推送投递状态和结果
- **运行日志** - 详细的执行日志和错误记录
- **状态跟踪** - 实时跟踪每个职位的投递状态

## 🏗️ 技术架构

### 前端层 (Next.js)
- **技术栈**: Next.js 15 + React 19 + TypeScript + Tailwind CSS
- **功能**: 任务创建、投递监控、配置管理界面
- **端口**: 6680 (开发) / 嵌入后端服务

### 后端层 (Spring Boot)
- **技术栈**: Spring Boot 3.5.7 + Java 21
- **功能**: 任务调度、数据库访问、状态推送
- **端口**: 8888
- **数据库**: SQLite (轻量级单文件数据库)

### 执行层 (Playwright)
- **技术栈**: Java + Playwright
- **功能**: 浏览器自动化、岗位抓取、智能投递
- **特点**: 支持多平台独立模块，自动处理登录和验证

## 🚀 使用流程

1. **环境配置** - 配置JDK 21、Node.js、Maven/Gradle
2. **AI配置** - 设置AI代理API Key和企业微信通知
3. **平台配置** - 配置各招聘平台的Cookie和筛选条件
4. **创建任务** - 选择平台、设置关键词和城市
5. **启动投递** - 运行自动化投递任务
6. **监控进度** - 通过Web界面实时查看投递状态

## ⚠️ 注意事项

- 需要关闭VPN/代理，主要针对国内招聘平台
- Boss直聘建议每日投递不超过150次
- 各平台需要定期更新Cookie保持登录状态
- 建议使用AI代理服务提高匹配精准度
- 本项目为开源项目，禁止商业化使用

## 📁 项目结构

```
get_jobs_plus/
├── front/                    # Next.js前端项目
├── src/main/java/com/getjobs/ # Spring Boot后端代码
├── doc/                      # 项目文档
├── gradle/                   # Gradle构建配置
└── src/main/resources/       # 静态资源和配置
```

## 🔧 配置文件

- **AI配置**: `.env`文件配置AI代理和企业微信
- **平台配置**: 通过Web界面配置各平台参数
- **数据库**: SQLite单文件存储，无需额外配置

## 📊 运行模式

- **开发模式**: 前后端独立启动，便于调试
- **生产模式**: 前端静态资源嵌入后端，一体化部署
- **独立Worker模式**: 单独运行自动化脚本

## 🎯 适用场景

- 批量投递简历，提高求职效率
- 精准筛选匹配职位，避免盲目投递
- 自动化处理重复性投递工作
- 实时跟踪投递状态和结果
